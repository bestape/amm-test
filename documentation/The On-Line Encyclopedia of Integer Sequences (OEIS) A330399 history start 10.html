Kyle MacLean Smith: "Mold & Cast Strings"
--->
My comments in A330390 revision #3 were a bit too simple. I forgot about the asymmetry of odd*even = even! So, the logic in that comment works for (1 + odd*sqrt(even)) and (1 + even*sqrt(even or odd)) approximations only. For (1 + odd*sqrt(odd)), the logic is not so simple and requires some choice. Here are my reasons in brief. 
--->
My paramount goal with these sequences is to connect Pythagoras's Theorem with Fibonacci Numbers using a = (x^(y-1))*(a*x + 2*b). When y = 2, (a^2) + (b^2) = c^2. When y = 2 and a = 1, 2*b = Metallic Mean Family. |Pi/y| is rotational change (conjecture) and the x^(y-1) polynomial degree is asymmetric, which means y = -1 associates with a 1/(x^2) degree two polynomial and a Pi/1 rotation. 
--->
For y = negative, the Fibonacci Spiral approximation is b(n<-y) = any positive ratio of integers, b(-y) = 2*b/a and b(n) = b(-y)*b(n+(y-1)) + b(n+y). Though b(n<-y) can equal any positive ratio of integers, given the y = -1 and a = 4 approximations of 1 + sqrt(2*b + 1), b(0) = 1/2 produces a powers of two denominator when 2*b = odd and b(0) = 1 produces a powers of two doubled up denominator when 2*b = even. If 2*b is odd, the denominator does not reduce the nominator and if 2*b is even, the denominator does reduce the nominator. For 2*b = 4 or 8 or 12... (sqrt(5) or sqrt(9) or sqrt(13)...), every nominator is reduced by the denominator and for 2*b = 2 or 6 or 10... (sqrt(3) or sqrt(7) or sqrt(11)...), every fourth nominator is reduced by the denominator, starting at b(2).
--->
This means that the Taylor Series for (1 + odd*sqrt(even)) and (1 + even*sqrt(even or odd)) approximations is (1 + (2*b - 2)*x)/(1 - 2*x - 2*b*(x^2)) and the Taylor Series for (1 + odd*sqrt(odd)) approximations is (1 + b*x - 2*(x^2) + ((b - 2)*b)*(x^3))/(1 - (2*b + 2)*(x^2) + (b^2)*(x^4)). These Taylor Series equations comport with the numerator of the Fibonacci pattern I'm using as a heuristic where, given y = -1 and a = 4, b(n) = (2*b/4)*b(n-2) + b(n-1). However, since the b(n) denominator always reduces the b(n) numerator if 2*b is divisible by a = 4, this fully reduced numerator-only sequence is the simple (1 - x^2)/(1 - x - (b/2)*(x^2)) Taylor Series.
--->
Despite the simplicity of the (1 - x^2)/(1 - x - (b/2)*(x^2)) Taylor Series, it has a different n value than its b(n) = (2*b/4)*b(n-2) + b(n-1) counterpart. The problem is the fully reduced Taylor Series starts with 1 + x but b(n) starts with (1 + 2*b/4*x). As such, it does not directly represent the Pythagorean relationship my work is trying to elicit. Therefore, I've decided to use the less simple (1 + b*x - 2*(x^2) + ((b - 2)*b)*(x^3))/(1 - (2*b + 2)*(x^2) + (b^2)*(x^4)) Taylor Series instead because it has the same n value as the unreduced b(n) = (2*b/4)*b(n-2) + b(n-1) for n>1 sequences and it works for all (1 + odd*sqrt(odd)) approximations. So, one Taylor Series for the more frequent (2*b = odd) and one Taylor Series for the less frequent (2*b = even), given the (1 + m*sqrt(n)) crossrefs context.
--->
(1 + b*x - 2*(x^2) + ((b - 2)*b)*(x^3))/(1 - (2*b + 2)*(x^2) + (b^2)*(x^4)) is interesting applied to (1 + sqrt(5)) approximations since (1 + 2*x - 2*x^2)/(1 - 6*x^2 + 4*x^4) also approximates ((1 + sqrt(5))/2) because the b(n) denominator is powers of two doubled up; A099603 is a nice twofer sequence. 
--->
So, for 2*b = (m^2 - 1) + ((m^2)*(n - 1)) approximations of (1 + m*sqrt(n)) = (1 + sqrt(2*b + 1)) = (1 + sqrt((m^2)*n)), (1 + b*x - 2*(x^2) + ((b - 2)*b)*(x^3))/(1 - (2*b + 2)*(x^2) + (b^2)*(x^4)) matches a(n) = (2*b + 2)*a(n-2) - (b^2)*a(n-4) for n>3 (2*b = even) and (1 + (2*b - 2)*x)/(1 - 2*x - 2*b*(x^2)) matches a(n) = 2*a(n-1) + 2*b*a(n-2) for n>1 (2*b = odd). Further, when 2*b is divisible by a, (1 - x^2)/(1 - x - b/2*x^2) matching a(n) = a(n-1) + b/2*a(n-2) for n>2 is a useful shortcut.
--->
In other words, this sequence is not (1 - x^2)/(1 - x - 11*x^2) and a(n) = a(n-1) + 11*a(n-2) for n>2, which is basically this sequence divided by powers of two doubled up, because of offset issues.
